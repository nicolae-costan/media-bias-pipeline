# Database Architecture Migration: ChromaDB to PostgreSQL + pgvector

This technical report details the migration of the database storage layer from ChromaDB to PostgreSQL with the `pgvector` extension. 

---

## 1. Migration Rationale

During the pipeline evolution, a structural misalignment occurred:
1. `article_embeddings.py` used a local vector database instance (**ChromaDB** in `./vector_db/`) to persist dense text representations.
2. `build_graph.py` (the topological KNN graph generator) queried **PostgreSQL** to fetch those same representations.

This decoupled architecture broke the pipeline end-to-end. Beyond this blocker, a pure vector store (like ChromaDB) lacks relational query capabilities. To link semantic vectors with structured metadata (such as publishing outlets, specific topics, annotator metrics, and links), a relational database with native vector-indexing capabilities is required.

By adopting **PostgreSQL** equipped with the **`pgvector`** extension:
*   We leverage powerful SQL capabilities (joins, indexing, aggregations, filtering).
*   We utilize high-performance vector indexing (**HNSW**) for fast nearest-neighbor searches.
*   All stages of the pipeline query a single, centralized database.

---

## 2. Relational Database Schema Design

To ensure optimal data normalization and separation of concerns, the storage layer is designed as a two-table schema:

### A. The `articles` Table
Stores static, human-annotated relational metadata extracted from the dataset CSVs. This information does not depend on neural network weight configurations.
*   `article_id` (TEXT, Primary Key): Unique hash identifier of the article.
*   `body` (TEXT): Raw body text of the article.
*   `outlet` (TEXT): Source news publisher (e.g., `"foxnews"`, `"nytimes"`).
*   `topic` (TEXT): Subject classification (e.g., `"abortion"`, `"immigration"`).
*   `type` (TEXT): Political leaning leaning (left/right/center).
*   `label_bias` (TEXT): Consensus bias judgment.
*   `news_link` (TEXT): URL source of the article.
*   `agreement` (FLOAT): Consensus weight ratio calculated via PySpark from the raw annotation study groups (SG1/SG2).

### B. The `article_embeddings` Table
Stores the continuous representations generated by our deep learning models. These vectors are updated whenever a model is retrained, but the corresponding metadata in the `articles` table remains untouched.
*   `article_id` (TEXT, Primary Key): Foreign Key referencing `articles.article_id` with `ON DELETE CASCADE`.
*   `embedding` (VECTOR(768)): Dense numerical vector representation of the article's semantics.
*   `emotion_scores` (FLOAT4[]): A 13-dimensional array representing the predicted probability distribution of the emotional heads.

### Hierarchical Navigable Small World (HNSW) Indexing
To scale vector similarity lookups to high volumes, a HNSW index is constructed on the `embedding` column using cosine operations:
```sql
CREATE INDEX IF NOT EXISTS article_embeddings_embedding_idx
ON article_embeddings
USING hnsw (embedding vector_cosine_ops);
```
This restricts the search space during KNN graph construction, turning $O(N)$ linear scans into fast $O(\log N)$ logarithmic lookups.

## 3. Automated Container Orchestration

A standardized container configuration is provided using **Docker Compose** (`docker-compose.yml`) using the official `pgvector/pgvector:pg16` image. The database listener is mapped to port `5433` to prevent port collisions.

To automate daemon state checks, a cross-platform Bash shell script is provided:
```bash
./start_db.sh
```
This script validates local Docker installations, verifies daemon responsiveness, starts the container, and reports ready status.

---

## 4. Quick Start: Setting Up a Fresh Workspace

To initialize the database layer on any Unix/Linux/macOS or WSL environment:

```bash
# 1. Start the PostgreSQL + pgvector container
chmod +x ./start_db.sh
./start_db.sh

# 2. Setup your local environment credentials file
cp GraphNeuralNetwork/.env_copy GraphNeuralNetwork/.env

# 3. Build the database schema (tables, foreign keys, HNSW indices)
python GraphNeuralNetwork/setup_db.py
```
